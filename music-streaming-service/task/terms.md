Техническое задание на разработку музыкального стримингового сервиса «HarmonyStream»

1. Введение

Документ определяет функциональные, технические и дизайн-требования к созданию веб-приложения для продажи подписок и прослушивания музыки – музыкального стримингового сервиса «HarmonyStream». ТЗ предназначено для разработчиков, дизайнеров и заказчика, фиксирует объём работ и критерии приёмки.

---

2. Общие сведения

2.1. Назначение системы

Сервис предназначен для:

· предоставления пользователям доступа к каталогу музыкальных треков (потоковое прослушивание);
· персонализированных рекомендаций на основе предпочтений и социального графа;
· создания, редактирования и совместного использования плейлистов в реальном времени;
· приобретения платных подписок (ежемесячных/годовых) с различными уровнями доступа;
· прослушивания музыки в офлайн-режиме (для премиум-подписчиков).

2.2. Конкурентные преимущества

· Социальное прослушивание: совместные сессии в реальном времени (синхронное воспроизведение, чат, обмен эмоциями);
· Совместные плейлисты с живым курсором: возможность одновременно редактировать плейлист с друзьями;
· Высокое качество звука: поддержка потоков 320 kbps (базовый тариф) и Lossless (премиум);
· Интеллектуальные рекомендации: гибридный подход (коллаборативная фильтрация + контентная фильтрация), настраиваемый пользователем (ползунки «знакомое–новое», «энергичное–спокойное»);
· Режимы «Фокус» и «Тренировка»: адаптивные плейлисты под текущую активность (определяется вручную или через интеграцию с календарём/фитнес-приложениями);
· Чистый интерфейс: отсутствие немузыкального контента (подкасты, видео), возможность включить режим «Только музыка».

2.3. Целевая аудитория

Сегмент А: Студенты (16–22 года)

· активные пользователи соцсетей, ищут музыку через друзей;
· высокая чувствительность к цене;
· нуждаются в социальных функциях (совместные плейлисты, сессии).

Сегмент Б: Молодые специалисты (23–35 лет)

· готовы платить за качество и персонализацию;
· используют музыку как фон для работы, спорта, отдыха;
· ценят экономию времени и неочевидные рекомендации.

География: приоритет – РФ и СНГ (русскоязычный интерфейс).

3. Структура сайта (карта страниц)

1. Главная страница (лендинг) – /
2. Каталог музыки – /catalog
3. Страница артиста/альбома – /artist/:id, /album/:id
4. Плеер – /player (или встроенный в любой раздел)
5. Тарифы и подписки – /pricing
6. Личный кабинет – /profile
   · Управление подпиской
   · Мои плейлисты
   · Настройки
7. Страница плейлиста – /playlist/:id (с поддержкой совместного редактирования)
8. Страница сессии прослушивания – /session/:id (синхронное прослушивание)
9. Авторизация / Регистрация – /login, /register
10. Статические страницы – /about, /terms, /privacy

Все страницы используют общие элементы:

· Шапка: логотип (ссылка на главную), навигационное меню, кнопки входа/регистрации (или аватар пользователя), глобальный поиск (на всех страницах, кроме плеера в полноэкранном режиме).
· Подвал: копирайт, контакты, ссылки на правовую информацию, соцсети.

4. Описание экранов (ключевые страницы)

4.1. Главная страница

Прототип (ссылка на Figma будет добавлена позже).

Обязательные элементы:

· Герой-блок с УТП (уникальное торговое предложение), призыв к бесплатному пробному периоду.
· Блок преимуществ (качество звука, социальные функции, офлайн-режим, умные рекомендации).
· Примеры популярных плейлистов / новинок.
· Блок тарифов (3 карточки: Базовый, Премиум, Семейный).
· Отзывы пользователей.
· Форма подписки на новости (email).

Поведение: при нажатии на «Начать бесплатный период» – переход на регистрацию с автоматическим активированным триалом.

---

4.2. Каталог музыки

Функции:

· Поиск по трекам, альбомам, артистам (с автодополнением).
· Фильтры по жанру, году, популярности.
· Сетка/список результатов с обложками, названием, исполнителем.
· Кнопка «Добавить в плейлист» / «Любимое» / «Поделиться».
· Быстрое воспроизведение при наведении/клике.

Особенность: на странице также отображаются виджеты:

· «Что слушают друзья»;
· «Популярные совместные плейлисты».

---
4.3. Страница плеера

Плеер может быть как отдельной страницей (/player), так и постоянно видимым элементом внизу экрана (аналог Spotify).

Функции:

· Обложка альбома, название трека, исполнитель.
· Кнопки управления: play/pause, предыдущий/следующий, повторить, перемешать, лайк.
· Прогресс-бар с возможностью перемотки.
· Регулятор громкости.
· Переключение качества звука (если доступно по подписке).
· Кнопка «Начать сессию» – создание комнаты для совместного прослушивания.
· Кнопка «Добавить в плейлист», «Поделиться».
· Отображение текста песни (опционально).

Режим «Только музыка»: кнопка, сворачивающая весь интерфейс, кроме плеера и обложки.

---

4.4. Страница плейлиста (совместного)

Уникальная фича:

· Отображение участников (аватарки).
· Курсоры редактирования – видно, кто в данный момент добавляет/удаляет трек.
· Чат внутри плейлиста.
· История изменений.

Действия:

· Добавление треков (поиск, перетаскивание).
· Изменение порядка.
· Удаление.
· Настройка доступа (публичный, по ссылке, только для друзей).

---

4.5. Личный кабинет

Разделы:

· Профиль: аватар, имя, email, дата регистрации.
· Подписка: текущий тариф, дата окончания, кнопка «Изменить/Отменить», история платежей.
· Мои плейлисты: список созданных, совместных, любимых треков.
· Настройки:
  · Качество звука по умолчанию (авто, 128, 320, Lossless);
  · Уведомления;
  · Приватность (кто видит мои плейлисты, активность);
  · Связанные аккаунты (VK, Google).
· Статистика: время прослушивания, любимые жанры, достижения (геймификация).

5. Требования к дизайну

5.1. Общая концепция

· Стиль: минималистичный, «чистый», современный (Neumorphism или Flat 2.0).
· Акцент: на обложках альбомов и типографике.
· Анимации: плавные, ненавязчивые (появление элементов, hover-эффекты, вращение пластинки).
· Цветовая схема:
  · Основной фон: тёмный (#121212) – стандарт музыкальных приложений, снижает нагрузку на глаза.
  · Акцентный цвет: #1DB954 (зелёный) – ассоциируется с музыкой, ростом.
  · Дополнительный акцент: #5352ED (синий) – для социальных элементов.
  · Текст: белый (#FFFFFF) и серый (#B3B3B3).
· Шрифты:
  · Заголовки: Montserrat, 800–900.
  · Основной текст: Poppins, 300–600.

5.2. Адаптивность

· Десктоп: от 1280px до 2560px.
· Планшеты: 768px–1024px (горизонтальная и вертикальная ориентация).
· Мобильные устройства: от 360px до 480px.

5.3. Фирменный стиль

· Логотип: текстовый «HarmonyStream» с нотой (предоставляется заказчиком).
· Иконки: использовать Font Awesome 6 или кастомный набор.

6. Технические требования

6.1. Платформа и стек технологий

· Frontend:
  · Фреймворк: React 18+ (или Vue 3) с TypeScript.
  · Стейт-менеджмент: Redux Toolkit / Pinia.
  · Стили: CSS Modules / Styled Components / SCSS.
  · Аудиоплеер: Howler.js / Web Audio API.
· Backend:
  · Язык: Node.js (NestJS) или Python (FastAPI).
  · База данных: PostgreSQL (основная), Redis (кэш, сессии, pub/sub).
  · API: REST + WebSockets (Socket.io) для real-time функций.
  · Аутентификация: JWT, OAuth 2.0 (VK, Google).
· Инфраструктура:
  · Контейнеризация: Docker, Docker Compose.
  · Хостинг: VPS / Облачные платформы (AWS, GCP, Yandex.Cloud).
  · CDN для статики и аудиофайлов.

6.2. Браузеры и устройства

· Десктопные браузеры: последние 2 версии Chrome, Firefox, Safari, Edge, Yandex Browser.
· Мобильные браузеры: Chrome for Android, Safari iOS.
· Адаптивная вёрстка: обязательна для всех разрешений.

6.3. Производительность

· Время загрузки главной страницы: < 2 секунд (Lighthouse).
· Оптимизация изображений (WebP), ленивая загрузка.
· Code splitting, tree shaking.
· Время отклика API: < 200 мс (95-й перцентиль).

6.4. Безопасность

· HTTPS на всех страницах.
· Защита от XSS, CSRF, SQL-инъекций.
· Хеширование паролей (bcrypt).
· Безопасное хранение платёжных данных (соответствие PCI DSS при интеграции с платёжными системами).

6.5. Интеграции

· Платёжные системы: ЮKassa / Stripe (подписки, разовые платежи).
· Email-рассылки: SendGrid / Mailchimp.
· Аналитика: Google Analytics 4 / Яндекс.Метрика.
· Социальные сети: VK API, Telegram API (шаринг, вход).
